# Деревья процессов и правил системы обслуживания

Ниже схемы в формате “событие → решение → действие → следующий ответственный”.  
Их можно использовать как основу для презентации руководству.

## 1. Общая заявка на обслуживание

```mermaid
flowchart TD
  A["СОЗДАНА ЗАЯВКА"] --> B{"Кто открыл?"}

  B -->|Администратор| C["Заявка сразу активна"]
  B -->|Менеджер отдела| C
  B -->|Работник| W1["Ожидает подтверждения менеджера"]
  B -->|Из жалобы уборки: поломка| C

  W1 --> W2{"Решение менеджера"}
  W2 -->|Одобрить| C
  W2 -->|Вернуть на исправление| W3["Работник дополняет данные / фото"]
  W2 -->|Отклонить| Z1["Заявка отменена"]
  W3 --> W1

  C --> D{"Тип заявки"}
  D -->|Здание / инфраструктура| F1["Определить категорию"]
  D -->|Транспорт / погрузчик| T1["Определить технику и простой"]

  F1 --> F2{"Кто будет выполнять?"}
  F2 -->|Техник| E1["Назначить техника"]
  F2 -->|Менеджер сам| E2["Менеджер принимает в работу"]
  F2 -->|Администратор / подрядчик| E3["Администратор ведет до закрытия"]

  T1 --> T2["Назначить приоритет и SLA"]
  T2 --> E1

  E1 --> G["Исполнитель принимает в работу"]
  E2 --> G
  E3 --> G

  G --> H{"Работа идет нормально?"}
  H -->|Да| I["Выполнение работ"]
  H -->|Нет| J["Ожидание с причиной"]

  J --> J1{"Причина ожидания"}
  J1 -->|Запчасти| J2["Ожидаем запчасти"]
  J1 -->|Поставщик| J3["Ожидаем поставщика"]
  J1 -->|Нет доступа| J4["Ожидаем доступ"]
  J1 -->|Нужно решение менеджера| J5["Ожидаем решение"]
  J1 -->|Техника не передана| J6["Особая ветвь: техника не у техника"]

  J2 --> I
  J3 --> I
  J4 --> I
  J5 --> I
  J6 --> TNE1["Ответственность переходит к менеджеру"]
  TNE1 --> TNE2["Менеджер передает технику"]
  TNE2 --> I

  I --> K["Исполнитель завершает работу"]
  K --> L{"Кто должен подтвердить результат?"}
  L -->|Открывший / менеджер| M["Ожидает подтверждения результата"]
  L -->|Если менеджер сам исполнитель| N["Ожидает финального закрытия"]

  M --> O{"Результат принят?"}
  O -->|Да| N
  O -->|Нет| G

  N --> P["Администратор закрывает окончательно"]
  P --> Q["Фиксация стоимости, поставщика, качества закрытия"]
  Q --> R["Заявка закрыта"]
```

## 2. Заявка от работника

```mermaid
flowchart TD
  A["РАБОТНИК ВИДИТ ПРОБЛЕМУ"] --> B["Входит по номеру работника и коду"]
  B --> C["Выбирает тип / описывает проблему"]
  C --> D["Обязательно прикладывает фото"]
  D --> E["Отправляет обращение"]
  E --> F["Статус: ожидает менеджера"]

  F --> G{"Менеджер проверяет"}
  G -->|Недостаточно данных| H["Вернуть работнику на исправление"]
  G -->|Дубль / не требуется| I["Отклонить"]
  G -->|Проблема реальная| J["Одобрить"]

  H --> K["Работник исправляет / дополняет"]
  K --> F

  I --> L["Обращение закрыто как отмененное"]

  J --> M["Менеджер задает категорию"]
  M --> N["Менеджер задает приоритет"]
  N --> O{"Куда направить?"}

  O -->|Технику| P["Создается активная заявка технику"]
  O -->|Администратору| Q["Передается администратору"]
  O -->|Менеджер сам обработает| R["Менеджер становится исполнителем"]
  O -->|Транспорт| S["Передается в транспортный поток"]

  P --> T["Обычный процесс ремонта"]
  Q --> T
  R --> T
  S --> T
```

## 3. Заявка по транспорту / погрузчику

```mermaid
flowchart TD
  A["ПРОБЛЕМА С ТЕХНИКОЙ"] --> B{"Кто открыл?"}
  B -->|Менеджер| C["Выбирает единицу техники"]
  B -->|Администратор| C
  B -->|Работник| B1["Сначала подтверждение менеджера"]
  B1 --> C

  C --> D["Указать тип простоя"]
  D --> D1{"Тип простоя"}
  D1 -->|Есть замена| E1["Простой есть, но работа отдела продолжается"]
  D1 -->|Некритично| E2["Техника работает ограниченно"]
  D1 -->|Критично, замены нет| E3["Высокий риск / высокий приоритет"]

  E1 --> F["Назначить SLA"]
  E2 --> F
  E3 --> F

  F --> G["Передать в пул техников / назначить техника"]
  G --> H{"Техник получил технику физически?"}

  H -->|Да| I["Техник принимает в работу"]
  H -->|Нет| J["Техник отмечает: техника не получена"]

  J --> K["Статус: ожидание"]
  K --> L["Причина: техника не передана"]
  L --> M["Система считает время ожидания"]
  M --> N["Уведомление менеджеру и администратору"]
  N --> O["Менеджер организует передачу техники"]
  O --> H

  I --> P{"Ремонт выполнен?"}
  P -->|Нет, нужна запчасть / поставщик / доступ| Q["Ожидание с причиной"]
  Q --> I
  P -->|Да| R["Техник добавляет результат / фото"]
  R --> S["Ожидает подтверждения менеджера"]

  S --> T{"Менеджер подтверждает?"}
  T -->|Нет| I
  T -->|Да| U["Ожидает закрытия администратора"]

  U --> V["Администратор фиксирует стоимость и качество"]
  V --> W["Заявка закрыта"]
```

## 4. Заявка по зданию / инфраструктуре

```mermaid
flowchart TD
  A["ПРОБЛЕМА В ЗДАНИИ"] --> B{"Кто открыл?"}
  B -->|Менеджер| C["Активная заявка"]
  B -->|Администратор| C
  B -->|Работник| D["Ожидает проверки менеджера"]
  B -->|Жалоба по зоне уборки: поломка| C

  D --> D1{"Решение менеджера"}
  D1 -->|Одобрить| C
  D1 -->|Вернуть| D2["Работник дополняет"]
  D1 -->|Отклонить| D3["Отмена"]
  D2 --> D

  C --> E["Категория: электричество / сантехника / HVAC / безопасность / IT / здание / другое"]
  E --> F["Приоритет: высокий / средний / низкий"]
  F --> G{"Кто обрабатывает?"}

  G -->|Техник здания| H["Назначить техника"]
  G -->|Менеджер сам| I["Менеджер принимает в работу"]
  G -->|Администратор / подрядчик| J["Администратор ведет задачу"]

  H --> K["Работа в процессе"]
  I --> K
  J --> K

  K --> L{"Есть блокировка?"}
  L -->|Нет| M["Работа выполнена"]
  L -->|Да| N["Ожидание: запчасти / поставщик / доступ / решение / бюджет / подрядчик"]
  N --> K

  M --> O{"Кто подтверждает результат?"}
  O -->|Открывший / менеджер| P["Подтверждение результата"]
  O -->|Администраторская задача| Q["Финальное закрытие"]

  P --> R{"Проблема решена?"}
  R -->|Да| Q
  R -->|Нет| K

  Q --> S["Администратор закрывает с затратами и качеством"]
  S --> T["Заявка закрыта"]
```

## 5. Закрытие заявки

```mermaid
flowchart TD
  A["ИСПОЛНИТЕЛЬ ЗАВЕРШИЛ РАБОТУ"] --> B{"Нужно подтверждение открывшего?"}

  B -->|Да| C["Ожидает подтверждения менеджера / открывшего"]
  B -->|Нет| F["Ожидает администратора"]

  C --> D{"Результат принят?"}
  D -->|Нет| E["Вернуть в работу с комментарием"]
  E --> A
  D -->|Да| F

  F --> G["Администратор проверяет историю"]
  G --> H["Указывает качество закрытия"]
  H --> H1{"Качество"}
  H1 -->|Полностью решено| I["Закрыть как решенное"]
  H1 -->|Временное решение| I
  H1 -->|Может повториться| I
  H1 -->|Нужна закупка / замена| I
  H1 -->|Нужен внешний подрядчик| I

  I --> J["Указать стоимость"]
  J --> K["Указать поставщика / подрядчика"]
  K --> L["Добавить примечание"]
  L --> M["Финальная подпись администратора"]
  M --> N["Статус: закрыта"]
```

## 6. Контур уборки: плановый обход

```mermaid
flowchart TD
  A["АДМИНИСТРАТОР СОЗДАЕТ ЗОНУ УБОРКИ"] --> B["Название, здание, этаж"]
  B --> C["Чек-лист зоны"]
  C --> D["Окна уборки: время + допустимое отклонение"]
  D --> E["Назначить ответственного уборщика"]
  E --> F["Зона активна"]

  F --> G{"Наступило окно уборки?"}
  G -->|Еще нет| H["Статус окна: запланировано"]
  G -->|Да| I["Статус окна: выполнить сейчас"]
  G -->|Просрочено, но окно еще открыто| J["Статус окна: в опоздании"]
  G -->|Окно закрыто| K["Статус окна: пропущено"]

  I --> L["Уборщик открывает зону"]
  J --> L

  L --> M["Отмечает чек-лист"]
  M --> N["Добавляет фото / комментарий при необходимости"]
  N --> O["Сохраняет обход"]
  O --> P["Статус окна: выполнено"]

  K --> Q["Администратор видит пропуск"]
  Q --> R["Попадает в аналитику уборки"]
  P --> R
```

## 7. Жалоба по уборке / зоне

```mermaid
flowchart TD
  A["СООБЩЕНИЕ О ПРОБЛЕМЕ В ЗОНЕ"] --> B{"Кто сообщил?"}

  B -->|Администратор| C["Жалоба сразу открыта"]
  B -->|Менеджер| C
  B -->|Работник уборки| D["Ожидает проверки администратора"]
  B -->|Анонимный QR-канал| D

  D --> E{"Администратор проверяет"}
  E -->|Отклонить| F["Жалоба отклонена"]
  E -->|Одобрить| C

  C --> G{"Тип проблемы"}
  G -->|Грязь| H["Открытая задача уборки"]
  G -->|Поломка / неисправность| I["Автоматически создать заявку по зданию"]

  H --> J["Уборщик / ответственный видит проблему"]
  J --> K["Проблема обработана"]
  K --> L["Отметить как решенную"]

  I --> M["Заявка идет в обычный процесс обслуживания"]
  M --> N["После ремонта закрывается администратором"]

  L --> O["Попадает в аналитику жалоб"]
  N --> O
  F --> O
```

## 8. Анонимный QR-канал

```mermaid
flowchart TD
  A["ПОЛЬЗОВАТЕЛЬ СКАНИРУЕТ QR ЗОНЫ"] --> B["Выбирает проблему"]
  B --> C{"Тип"}
  C -->|Грязь| D["Жалоба на уборку"]
  C -->|Поломка| E["Жалоба на неисправность"]

  D --> F["Обязательное фото"]
  E --> F
  F --> G["Комментарий необязателен"]
  G --> H["Отправка"]
  H --> I{"Проверка частоты отправки"}

  I -->|Слишком часто| J["Отправка блокируется временно"]
  I -->|Разрешено| K["Статус: ожидает проверки администратора"]

  K --> L{"Администратор"}
  L -->|Одобрить грязь| M["Открытая жалоба уборки"]
  L -->|Одобрить поломку| N["Создать заявку по зданию"]
  L -->|Отклонить| O["Жалоба отклонена"]
```

## 9. Документы и плановое обслуживание техники

```mermaid
flowchart TD
  A["ЕДИНИЦА ТЕХНИКИ В СИСТЕМЕ"] --> B["Хранятся документы и даты"]
  B --> C["Хранятся плановые обслуживания"]

  B --> D{"Срок документа подходит?"}
  D -->|Нет| E["Нормальный статус"]
  D -->|Скоро истекает| F["Предупреждение"]
  D -->|Истек| G["Критический риск документа"]

  F --> H["Уведомление администратору"]
  G --> H

  C --> I{"Плановое обслуживание близко?"}
  I -->|Нет| J["Ожидание следующей даты"]
  I -->|Да| K["Уведомление: ТО скоро"]
  I -->|Просрочено| L["Уведомление: ТО просрочено"]

  K --> M["Назначить выполнение"]
  L --> M
  M --> N["Техник выполняет ТО"]
  N --> O{"Есть неисправности?"}

  O -->|Нет| P["Записать выполненное ТО"]
  O -->|Да| Q["Автоматически открыть ремонтную заявку"]
  Q --> R["Обычный процесс обслуживания"]
```

## 10. Назначение водителей на технику

```mermaid
flowchart TD
  A["МЕНЕДЖЕР ХОЧЕТ НАЗНАЧИТЬ ВОДИТЕЛЯ"] --> B{"Тип действия"}

  B -->|Добавить водителя| C["Проверить слот / смену"]
  B -->|Перенести водителя| D["Проверить целевую технику и смену"]
  B -->|Доступ / чип| E["Запросить доступ"]

  C --> F{"Нужно подтверждение администратора?"}
  D --> F
  E --> F

  F -->|Нет| G["Изменение применяется сразу"]
  F -->|Да| H["Создается запрос администратору"]

  H --> I{"Администратор решает"}
  I -->|Одобрить| J["Водитель назначен / перенесен"]
  I -->|Отклонить| K["Запрос отклонен"]

  J --> L["Менеджер получает уведомление"]
  K --> L
  G --> M["История действия сохранена"]
  L --> M
```

## 11. Короткое правило ответственности

```mermaid
flowchart TD
  A["У заявки есть текущий статус"] --> B{"Кому принадлежит следующий шаг?"}

  B -->|Новая / в работе / ожидание| C["Исполнитель: техник / менеджер / администратор"]
  B -->|Ожидает подтверждения работника| D["Менеджер отдела"]
  B -->|Ожидает подтверждения результата| E["Открывший / менеджер"]
  B -->|Ожидает финального закрытия| F["Администратор"]
  B -->|Техника не передана технику| G["Менеджер должен передать технику"]
  B -->|Закрыта / отменена| H["Действий не требуется"]
```
